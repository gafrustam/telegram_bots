## §12. Mini App — специфические сценарии

### 12.1 — Разрешение микрофона не выдано
**Платформа:** Mini App (Telegram)
**Действия:** пользователь нажимает кнопку записи, браузер запрашивает доступ, пользователь отклоняет
**Ожидается:** toast «Microphone access denied: …»; кнопка записи остаётся доступной для повторной попытки

**Результат:** ✅ `navigator.mediaDevices.getUserMedia` пробрасывает ошибку → `showToast`

---

### 12.2 — Telegram BackButton
**Платформа:** Mini App
**Действия:** нажать системную кнопку «Назад» Telegram
**Ожидается:** переход по `backMap`: recording → topic_selection → menu; на Home — BackButton скрывается

**Результат:** ✅ `TgApp.BackButton.onClick(navigateBack)`

---

### 12.3 — Смена темы Telegram
**Платформа:** Mini App
**Действия:** пользователь меняет тему в Telegram (тёмная / светлая)
**Ожидается:** CSS-переменные обновляются, контраст сохраняется

**Результат:** ⚠️ CSS-переменные применяются только при загрузке. Динамическое обновление темы требует подписки на `themeChanged` событие (не критично)

---

### 12.4 — Mini App открыт с новой версии Telegram без SDK
**Действия:** пользователь использует старый клиент без `window.Telegram.WebApp`
**Ожидается:** приложение работает в браузер-режиме (fallback CSS-переменные), без initData

**Результат:** ✅ `const TgApp = window.Telegram?.WebApp || null` + fallback

---

### 12.5 — initData HMAC невалиден
**Действия:** передан некорректный X-Telegram-Init-Data
**Ожидается:** сервер игнорирует, статистика не привязывается к пользователю (анонимная сессия)

**Результат:** ✅ `_validate_telegram_init_data` возвращает None

**Ожидается:** тот же `session_token` из localStorage; сессии на сервере привязаны к токену (TTL 1 час)

---

### 13.2 — Разрешение микрофона в браузере
**Платформа:** Browser
**Действия:** первый клик на запись → браузер спрашивает доступ
**Ожидается:** если отклонено → toast; если разрешено → запись стартует

**Результат:** ✅ аналогично §12.1

---

### 13.3 — Неподдерживаемый MIME-тип записи
**Платформа:** Browser (Safari, Firefox)
**Действия:** запись в браузере без поддержки `audio/webm`
**Ожидается:** `_pickMimeType()` пробует `audio/ogg;codecs=opus` → `audio/ogg`; сервер определяет формат по magic bytes

**Результат:** ✅ fallback цепочка в recorder.js + `_detect_audio_ext` в server.py

---

### 13.4 — Анонимность: нет статистики по user_id
**Платформа:** Browser (без Telegram)
**Действия:** нажать «📊 My Statistics»
**Ожидается:** `{"sessions": [], "summary": null}` → сообщение «No completed sessions yet.»

**Результат:** ✅ сервер возвращает пустой ответ для неавторизованных

---

### 13.5 — Прямой URL на страницу статистики
**Платформа:** Browser
**Действия:** открыть `gafrustam.ru/ielts/` → тапнуть статистику
**Ожидается:** нет редиректа на вход; приложение одностраничное, роутинг через JS

**Результат:** ✅ SPA, nginx proxy → FastAPI → `GET /` → index.html

## §15. Сознательно не исправленные замечания

| Замечание | Почему оставлено |
|-----------|-----------------|
| Стикеры/фото молча игнорируются (Bot) | Aiogram по умолчанию; catch-all усложнит код без пользы |
